# TZS address extractor

Веб-приложение на Streamlit для извлечения из Excel-файлов двух рабочих полей:

- `Населённый пункт`
- `Номер дома, улица`

Версия интерфейса: `v1.13.6`.

## Возможности

- Автоматически определяет столбец с полным адресом.
- Поддерживает стандартные адреса, обратную запись улиц, километровые адреса, МКАД/ЦКАД, автодороги, строения, владения, сооружения и участки.
- Возвращает компактную адресную часть с лимитом 30 символов.
- Сохраняет исходную строку для проверки.
- Формирует готовый Excel-файл для скачивания.
- Ведёт постоянный журнал использования без сохранения адресов из файлов.

Приложение работает локально, без внешних API и без LLM.

## Запуск

### Docker

```bash
docker compose up -d --build
```

Приложение будет доступно на `http://localhost:8501`.

Остановить:

```bash
docker compose down
```

### Python

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
streamlit run app.py
```

## Выходной файл

После обработки приложение создаёт Excel-файл с колонками:

| Колонка             | Описание                                                                  |
| ------------------- | ------------------------------------------------------------------------- |
| `Населённый пункт`  | Самое подходящее значение населённого пункта или административного уровня |
| `Номер дома, улица` | Компактная адресная часть: дом/строение/км и улица/дорога/ориентир        |
| `Исходная строка`   | Полный исходный адрес                                                     |

## Журнал использования

События записываются в `logs/usage.jsonl`. Каталог подключён к контейнеру
как volume, поэтому журнал сохраняется после перезапуска и пересборки.

Журнал содержит:

- дату и время в UTC;
- идентификатор сессии;
- имя и размер загруженного файла;
- запуск и завершение обработки;
- количество строк и длительность;
- скачивание результата;
- тип ошибки, если обработка завершилась неуспешно.

Содержимое адресов в журнал не записывается.

Сводка по дням:

```bash
docker exec address-parser python scripts/usage_summary.py /app/logs/usage.jsonl
```

Последние события:

```bash
tail -n 50 logs/usage.jsonl
```

## Безопасность

- Файлы обрабатываются в памяти текущей сессии.
- В журнале сохраняются только имя файла и технические метрики; адресные строки не сохраняются.
- Внешние API не используются.
- Ключи и пользовательские данные не требуются.
